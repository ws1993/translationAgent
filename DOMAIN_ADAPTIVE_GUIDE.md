# 领域自适应翻译功能说明

## 功能概述

领域自适应翻译是在现有"快速模式"和"专业模式"基础上新增的第三种翻译模式,通过 AI Agent 自动识别文本领域并动态生成专业术语表,实现更精准的专业翻译。

## 核心特性

### 1. 智能领域识别
- 自动分析文本内容,识别所属专业领域(技术、法律、医疗、金融等)
- 支持子领域细分识别(如:技术 → 人工智能、法律 → 合同法)
- 提供置信度评分和判断理由

### 2. 动态术语生成
- 根据文本内容和识别的领域,即时生成专业术语双语对照表
- 包含术语使用场景和翻译注意事项
- 无需预先建立术语库,零维护成本

### 3. 领域增强翻译
- 基于领域知识和术语表进行三段式翻译(直译 → 问题分析 → 意译)
- 强制遵守术语表中的标准译法,确保专业术语准确性和一致性
- 从领域专业角度进行问题识别和改进

### 4. 实时进度可视化
- 5 个阶段可视化展示:领域识别 → 术语生成 → 初步翻译 → 问题分析 → 精准优化
- 每个阶段显示实时进度和关键结果
- 领域识别结果和术语数量即时反馈

### 核心流程

```
用户输入文本
    ↓
选择"领域自适应"模式
    ↓
[阶段1] 领域识别 Agent
    - 分析文本特征
    - 输出: { primaryDomain, subDomain, confidence, reasoning }
    ↓
[阶段2] 术语生成 Agent  
    - 根据领域和文本内容生成术语表
    - 输出: [{ source, target, context, notes }]
    ↓
[阶段3-5] 领域增强三段式翻译
    - 使用领域提示词 + 术语约束
    - 直译 → 问题分析(领域视角) → 意译
    ↓
返回完整翻译结果 + 领域信息 + 术语表
```

### LLM 调用次数

| 阶段 | 调用次数 | 说明 |
|------|---------|------|
| 领域识别 | 1 次 | 零样本分类,返回 JSON |
| 术语生成 | 1 次 | 基于领域和文本,返回 JSON 数组 |
| 直译 | 1 次 | 流式输出 |
| 问题分析 | 1 次 | 非流式 |
| 意译 | 1 次 | 流式输出 |
| **总计** | **5 次** | 比专业模式多 2 次 LLM 调用 |

## 使用方法

### 步骤 1: 选择模式

在翻译页面选择"领域自适应"模式:

```
翻译模式: [快速模式] [专业模式] [领域自适应] ← 选择此项
```

### 步骤 2: 输入文本

在原文区域输入需要翻译的专业文本,例如:

```
深度学习模型通过反向传播算法优化损失函数,使用梯度下降更新神经网络的权重参数。
卷积神经网络(CNN)特别适合处理图像数据,而循环神经网络(RNN)更擅长序列建模。
```

### 步骤 3: 开始翻译

点击"开始翻译"按钮,系统会自动弹出进度弹窗,展示 5 个阶段的实时进展:

**阶段 1: 领域识别**
```
识别结果: 技术 · 人工智能 (置信度: 95%)
```

**阶段 2: 术语生成**
```
生成术语: 12 个专业术语
示例:
- 深度学习 → Deep Learning
- 反向传播 → Backpropagation  
- 卷积神经网络 → Convolutional Neural Network (CNN)
```

**阶段 3-5: 翻译过程**
- 初步翻译:基于术语表直译
- 问题分析:从 AI 领域专业角度识别问题
- 精准优化:生成符合领域规范的译文

### 步骤 4: 查看结果

翻译完成后,在译文区域查看最终翻译结果。

## 实现细节

### 领域识别 Prompt 设计

```
你是一位专业的文本领域分类专家,能够准确识别文本所属的专业领域。

请分析以下文本,识别其所属的专业领域。

输出格式(JSON):
{
  "primaryDomain": "主要领域",
  "subDomain": "子领域(可选)",
  "confidence": 0.95,
  "reasoning": "判断理由"
}
```

### 术语生成 Prompt 设计

```
你是一位{领域}领域的专业术语专家,精通中英文专业术语对照。

请根据文本内容,生成该文本中涉及的{领域}专业术语的双语对照表。

要求:
1. 识别专业术语、固定表达、技术名词
2. 给出准确的对应翻译
3. 优先考虑该领域的标准译法和行业惯例

输出格式(JSON数组):
[
  {
    "source": "源语言术语",
    "target": "目标语言翻译",
    "context": "使用场景(可选)",
    "notes": "翻译注意事项(可选)"
  }
]
```

### 领域提示词构建

根据识别的领域信息和生成的术语表,动态构建如下格式的提示词:

```markdown
## 领域背景
本文属于【技术】领域,细分领域为【人工智能】。
判断依据:文本包含大量AI、机器学习相关术语,讨论模型训练和算法优化

## 专业术语表(必须严格遵守)
1. 深度学习 → Deep Learning (AI技术分支)
2. 反向传播 → Backpropagation
   注意:不翻译为 Backward Propagation
3. 卷积神经网络 → Convolutional Neural Network (CNN)

## 翻译要求
- 严格遵守上述术语表中的译法,确保专业术语翻译的准确性和一致性
- 符合技术领域的表达习惯和专业规范
- 保持原文的专业性和严谨性
```

## 最佳实践

### 1. 适用场景

**推荐使用**:
- 技术文档、API 文档、技术规范
- 学术论文、研究报告
- 法律合同、医疗报告
- 金融分析、商业计划书
- 任何包含大量专业术语的文本

**不推荐使用**:
- 日常对话、口语化文本 → 使用"快速模式"
- 文学作品、散文诗歌 → 使用"专业模式"
- 非常短的文本(1-2句话) → 使用"快速模式"

### 2. 性能考虑

- 领域识别和术语生成各需约 3-10 秒(取决于 LLM 响应速度)
- 建议文本长度:100-2000 字
- 对于超长文本(>3000 字),可考虑分段翻译

### 3. 成本优化

领域自适应模式比专业模式多 2 次 LLM 调用:
- 如果使用 GPT-4,成本约为专业模式的 1.67 倍
- 如果使用 GPT-3.5/Claude,额外成本可忽略不计
- 建议配置时优先使用高性能模型以获得更准确的领域识别

## 技术亮点

### 1. 零维护成本

- 无需预先建立术语库
- 无需人工标注领域
- 术语即时生成,不持久化存储
- 自动适配不同领域

### 2. 参考前沿研究

实现参考了以下学术研究成果:

- **CRAT** (Causality-Enhanced Reflective and Retrieval-Augmented Translation): 因果增强反思 + 术语图谱
- **HiMATE** (Hierarchical Multi-Agent Framework): 多智能体协作翻译
- **ReflectMT**: 反思式机器翻译
- **Domain Adaptation for NMT**: 领域自适应神经机器翻译

### 3. 多智能体协作

- 领域分类 Agent: 专注于领域识别
- 术语专家 Agent: 精通专业术语对照
- 翻译 Agent: 基于领域知识进行翻译
- 问题分析 Agent: 从领域视角进行质量评估

## 故障排查

### Q: 领域识别不准确怎么办?

A: 领域识别依赖 LLM 的理解能力,如果识别不准:
1. 检查文本是否包含足够的领域特征词
2. 尝试使用更强大的模型(如 GPT-4)
3. 文本太短可能导致识别困难,建议至少 50 字以上

### Q: 术语生成数量很少或为空?

A: 可能原因:
1. 文本中专业术语确实较少
2. LLM 响应格式解析失败(检查控制台日志)
3. 即使术语为空,翻译流程仍会继续,不影响最终结果

### Q: 翻译速度较慢?

A: 正常现象:
- 领域自适应模式需要 5 次 LLM 调用
- 前两个阶段(领域识别 + 术语生成)是额外开销
- 总耗时通常在 30-60 秒(取决于 LLM 服务响应速度)

### Q: 术语约束没有生效?

A: 检查:
1. 确认术语表已成功生成(在进度弹窗中可见)
2. 查看控制台日志,确认术语表已注入到提示词中
3. LLM 可能偶尔不遵守约束,这是模型本身的限制

## 未来改进方向

1. **术语库持久化**(可选):允许用户保存和复用生成的术语表
2. **领域预选**:支持用户手动指定领域,跳过识别阶段
3. **术语编辑**:翻译前允许用户审查和修改生成的术语表
4. **多模型协作**:不同阶段使用不同模型(如识别用 GPT-3.5,翻译用 GPT-4)
5. **翻译记忆集成**:结合 WebDAV 存储历史术语,提升一致性

## 总结

领域自适应翻译模式是对现有翻译功能的重要增强,特别适合处理专业文档翻译场景。通过智能领域识别和动态术语生成,在零维护成本的前提下显著提升专业术语的准确性和一致性,是处理技术、法律、医疗等专业文本的理想选择。
